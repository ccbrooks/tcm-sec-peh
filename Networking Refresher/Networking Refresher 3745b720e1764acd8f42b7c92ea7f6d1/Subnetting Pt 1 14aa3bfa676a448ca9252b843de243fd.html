Subnetting Pt. 1

ifconfig

INET = IPv4

netmask = subnet mask or subnet

Subnets are just like IP addresses in that each section is 8 bits, up to 255 in value of ones and zeroes

255.255.255.0 is known as a /24 or wack24 network and is standard and common

  • used in households and small businesses
  • allows a lot of hosts (256)
  • called /24 because when you count the number of “1”s in the address, there are 24!

As more bits are “on” in the fourth part of the subnet, less hosts are available to users of the subnet

Devices in your house connect through NAT out

To write a subnet, you can use the idea of number of bits turned on by creating a chart and using it to determine the subnet value

We always subtract 2 from the host because we can have that many possible hosts in a given IP range, since we have already used Network and Broadcast space

The second line works because we see that we have /28 in the 240 subnet mask spot. When we put the range from Network to Broadcast, we see that “ “.0 to “ “.15 leaves 16 spaces, and since those two are allocated, we have room for 14 hosts!

Third line uses the same host number, but changes the ranges from 16 to 31, using the same 16 spots in memory

For the /23 system, we actually use “x.x.0.0” instead of “x.x.1.0” because we can use hosts from the first and second set of bits to make it the 512 we are allocated

To get to 192.168.2.0, we could use the same subnet mask of 255.255.254.0 with 510 hosts. The only difference would be that we would have a network IP and broadcast ID of 192.168.2.0 and 192.168.3.255
