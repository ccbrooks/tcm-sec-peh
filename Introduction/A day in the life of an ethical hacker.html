A day in the life of an ethical hacker

A pen tester’s day-to-day:

perform assessment

write a report

debrief

Def external pen test - looking at an org’s security from the outside

Trying to hack in from another location

The methodology focuses on Open-Source Intelligence (OSINT) gathering

Typically takes 32-40 hrs and then 8-16 hrs for reporting

How can we use “basic” info against org (who are the employees, what breaches are they involved in, etc.)

Cheaper than other forms of assessments (good way for org to “test you out”

Def internal pen test - assessing an org’s security from the inside

Breach perimeter (phishing email, break into physical building, send laptop to company, etc)

The methodology focuses on active directory attacks

Def active directory attack -

Typically takes 32-40 hrs and then 8-16 hrs for reporting

*More websites to networks

Def web app pen test - assessing an org’s web app security

The methodology focuses on web-based attacks and the OWASP testing guidelines

Def OWASP - open web application security project

Typically takes 32-40 hrs and then 8-16 hrs for reporting

  • OWASP Top 10 attacks
    • OWASP Top 10:2021

Def wireless pen test - assessing an org’s wireless network security

The methodology depends on network being used:

if guest network → get on guest network and test segmentation

if pre-shared key → test key for password strength (WPA2-PSK)

enterprise-based → (WPA2 Enterprise)

Typically takes 4-8 hrs per SSID and then 2-4 hrs for reporting

Need wireless adapter

Def physical pen test - assessing an org’s physical security and/or end-user training

The methodology depends on tasks and goals

Going on site and trying to break into building (picking locks, stealing badges, etc)

Typically lasts 16-40 hrs and then 4-8 hrs for reporting

Def social engineering - creating a campaign against an org

phishing - sending emails with links and seeing who clicks it

vishing - calling people and saying we’re from IT (getting password or code from clients

smishing - sending texts to users and waiting for potential responses

Typically lasts 16-40 hrs and then 4-8 hrs for reporting

More types of pen testing:

mobile pen testing

IoT pen testing

Red Team Engagements - sneak in creatively

Purple Team Engagements - tabletop exercises with blue team (defensive) members

*Weakest link in any org is a human

To be a strong pen tester, need:

  1. technical skills
  1. writing/communicating skills
  1. presentation skills (technical and non-technical)
  1. CLEAR recommendations (clear and technical)

Report within a week (high level and technical)

Executive summary (non-technical)

Debrief process:

Walk client through report findings (non-technical and technical)

Gives clients opportunity to ask questions, challenge findings, and report concerns
